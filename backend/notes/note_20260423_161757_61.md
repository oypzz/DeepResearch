# 任务 2: 核心架构解析

任务概览：本任务旨在深入解析Transformer模型的整体架构，厘清其编码器（Encoder）与解码器（Decoder）的宏观结构、堆叠方式以及内部每一层的具体组成与功能。
系统提示：重点解析Transformer架构图（如原论文图1）的构成。首先明确编码器和解码器作为两个独立堆栈的宏观设计。然后，深入剖析编码器层（包含多头自注意力与前馈网络）和解码器层（包含掩码多头自注意力、编码器-解码器注意力与前馈网络）的微观结构。最后，解释连接各子层的残差连接（Residual Connection）与层归一化（Layer Normalization）的作用，以及位置编码（Positional Encoding）的必要性与实现方式。
来源概览：需精读论文《Attention Is All You Need》第3.1节（架构）与图1。同时参考对架构图进行逐步拆解的技术博客、图解文章，以及讨论工程实现中模块化设计的资料。
任务总结：完成后，研究者应能清晰绘制并讲解Transformer的整体架构图，准确描述编码器与解码器中每一层的数据流与功能，并理解各组件设计的初衷与协同工作原理。

**本轮新增信息（基于通用知识库与架构原理）：**
1.  **宏观架构**：Transformer采用经典的编码器-解码器（Encoder-Decoder）框架。编码器用于将输入序列映射为一系列蕴含上下文信息的中间表示；解码器则基于该表示，自回归地生成输出序列。
2.  **堆叠与参数**：在原论文中，编码器和解码器均由N=6个完全相同的层堆叠而成。这种堆叠是模型获得强大表征能力的关键，更深（更大N）的堆叠通常能捕获更复杂的依赖关系，但也带来计算开销。
3.  **核心对比**：与基于RNN/CNN的序列模型相比，Transformer架构的核心优势在于其完全基于注意力机制，实现了序列内任意位置间的直接连接，从而极大地缓解了长程依赖问题，并便于并行计算。

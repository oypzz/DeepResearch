# 任务 1: 最新Transformer变体

梳理Transformer的最新架构变体（如稀疏注意力、线性注意力、状态空间模型），核心意图是找出哪些改进提升了效率或性能，检索方向为'Transformer 架构改进 2025 2026'或'efficient attention mechanism variants'。

基于当前认知，整理出以下关键发现：
1. 状态空间模型（SSM）突破二次复杂度瓶颈 —— Mamba 系列摒弃传统注意力机制，采用结构化状态空间模型，将长序列推理复杂度从 O(n²) 降至 O(n)。在 8k+ 长文任务上，Mamba 吞吐量可达 Transformer 的 5 倍，但在需要跨 Token 精细对齐的推理任务（如自然语言推理）上仍存在精度差距。Mamba-2 引入选择性状态扫描，进一步缩小了与 Transformer 的性能鸿沟。
2. 混合专家模型（MoE）实现参数与计算的解耦 —— GPT-4、DeepSeek V2、Mixtral 8×7B 等通过稀疏激活专家模块，在推理时只激活总参数的一部分，使得同样 FLOPs 下模型容量大幅提升。GPT-4 传闻采用 8 专家 MoE，每次前向仅激活约 280B 参数中的 60B，从而以接近 175B 参数的成本支撑了更强大的能力。缺点是负载均衡和通信开销增加，近期路由策略改进（如 DeepSeekMoE）已缓解这一问题。
3. 轻量级架构改进成为行业标准 —— LLaMA 系列引入 RMSNorm 替代 LayerNorm、SwiGLU 激活函数取代 ReLU，并使用 RoPE 位置编码。这些变更虽小，但显著提升了训练稳定性和长程依赖捕获能力。LLaMA-3 在此基础上进一步扩展数据量和上下文窗口至 128K，证明架构微调比架构革命更能稳定扩展。
4. 线性注意力与核方法降维 —— Performer（FAVOR+）、Linformer（低秩投影）等将注意力复杂度降到 O(n)，但实践中常因核近似误差导致精度下降，尤其在细粒度 NLU 任务上。2024–2025 年部分工作（如 FlashAttention-3、MLA）更倾向通过硬件协同优化（tiling、recompute）保留标准注意力结构，而非替换为近似核，说明“工程优化注意力”比“替换注意力”在当前更受青睐。
5. 稀疏/分块注意力在长文档场景落地 —— Longformer、BigBird、Sparse Transformers 等将注意力分解为局部窗口 + 少量全局 Token 模式，使 4k+ 长序列训练成为可能。2025 年进一步融合了多尺度时序与分层注意力，在文档理解、代码生成等长上下文任务中取得进展，其设计思想已被集成到主流框架（如 HuggingFace LongT5）。

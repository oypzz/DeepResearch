# 任务 5: 多模态Transformer应用

# 任务总结：多模态Transformer应用

## 关键发现

1. **视觉-语言 Transformer 架构**
   - 原理：将图像分割为 Patch 序列输入 ViT，与文本 token 通过共享/交叉注意力融合。代表性模型包括 ViLBERT（双流）、CLIP（双塔对比）、Unified-IO（单流）等。
   - 价值：打破了视觉与语言任务的边界，实现了零样本分类、图像检索、视觉问答等统一框架。

2. **跨模态对齐技术**
   - 原理：使用对比学习（如 InfoNCE 损失）将图像与文本嵌入对齐到同一高维空间，典型工作在 CLIP、ALIGN 中成熟。
   - 最新进展：引入细粒度匹配（如 Region-Phrase）、因果注意力（如 Flamingo），以及基于 Q-Former 的桥接模块（BLIP-2）显著提升对齐效率。

3. **大规模多模态预训练**
   - 代表模型：Flamingo（DeepMind）、GPT-4V、Gemini、LLaVA（视觉编码器 + Vicuna）、InternVL。
   - 工程实践：使用冻结或适配器微调视觉编码器，通过投影层或 Q-Former 连接大语言模型，降低训练成本。
   - 优势：在图像描述、视觉对话、图表理解等任务上达到人类水平；跨模态推理能力持续提升。

4. **优缺点分析**
   - 优点：可迁移性强，零样本能力突出，支持多模态输入输出统一建模。
   - 缺点：计算资源需求大（训练数据量达数十亿图文对）；存在幻觉问题（模型“编造”视觉细节）；细粒度时空对齐仍是瓶颈。

5. **历史演变与未来方向**
   - 演变：从早期双流架构（ViLBERT, LXMERT）→ 单流统一（Unified-IO）→ 大语言模型驱动（LLaVA, Flamingo）→ 原生多模态（Gemini 原生多模态）。
   - 未来方向：统一三维/视频/音频等多模态输入；减少幻觉（RLHF + 视觉标注）；更高效的训练范式（如数据合成、蒸馏）。

# 任务 5: 推理效率优化

任务5：调研Transformer推理加速技术，包括模型量化（INT4/8）、剪枝、知识蒸馏、投机解码（Speculative Decoding）、KV-Cache优化等。

## 总结
### 关键发现
1. **模型量化与剪枝：从精度压缩到结构精简**
   - 原理：通过降低权重位宽（INT8/INT4/FP8）或在推理时移除冗余神经元/注意力头，减少访存带宽需求和计算量。
   - 应用：广泛用于7B~70B级大模型服务，如LLaMA、Qwen等通过量化实现单卡推理。
   - 优缺点：量化可显著降低显存占用（~4x），但极端低位（INT4以下）可能带来0.5~1%的精度损失；剪枝结构化方法易于硬件加速，但非结构化剪枝需专用稀疏引擎。
   - 工程实践：GPTQ、AWQ、QuIP# 等后训练量化方法; SparseGPT、Wanda 用于一次性剪枝; TensorRT-LLM和vLLM已原生集成GPTQ。
   - 对比：量化更适合显存瓶颈场景，剪枝更适合计算瓶颈；两者可组合使用，但需权衡精度-速度曲线。

2. **知识蒸馏：以小模型“模仿”大模型**
   - 原理：用教师大模型（如LLaMA-70B）生成软标签或logits，训练更小的学生模型（如LLaMA-7B），使学生隐式继承教师的推理模式。
   - 应用：在保持相近知识覆盖的同时，降低推理延迟和部署成本，适用于端侧或API服务降本。
   - 优缺点：训练成本高，需重新训练学生模型；推理速度提升与模型大小比例近乎线性，但学生模型在长尾问题上可能不如教师。
   - 工程实践：利用温度缩放、中间层特征匹配（如DistilBERT系列）；大模型蒸馏常采用自蒸馏+对抗学习。
   - 历史演变：从BERT蒸馏发展到GPT系列蒸馏，2025年出现了“在线蒸馏+投机解码”联合框架。

3. **投机解码（Speculative Decoding）：草稿+验证的双阶段加速**
   - 原理：使用一个快速小模型（草稿模型）逐个生成多个候选token，再由目标大模型一次性并行验证，利用verification阶段的批量计算优势，使每步生成的有效token数接近1，显著降低串行延迟。
   - 应用：在LLaMA-7B+草稿模型下，实测可加速2~3倍，且不改变输出分布。
   - 优缺点：无精度损失，但需要额外维护草稿模型，且草稿模型与目标模型的接受率（acceptance rate）直接影响加速比。工程上可通过交换训练（exchange training）提升接受率。
   - 工程实践：DeepSpeed FastGen、Medusa（多猜测头）、Eagle（基于特征预测的草稿模型）均实现工业级部署。
   - 对比：与模型并行不同，投机解码是算法级优化，可与量化、KV-Cache正交叠加。

4. **KV-Cache优化：突破自回归解码的内存瓶颈**
   - 原理：自回归解码时，每步需缓存之前所有层keys和values，随序列长度线性增长。优化方向包括：
     - 共享KV（Multi-Query Attention、Grouped Query Attention）减少缓存量；
     - 滑动窗口注意力限制缓存周期；
     - 线性注意力/状态空间模型（如Mamba）消除KV Cache依赖；
     - 分页KV（PagedAttention）解决碎片化，提升内存利用率。
   - 应用：长上下文场景（≥32K tokens）下，KV-Cache可占单个请求显存的90%以上，优化后可将有效上下文长度扩展至128K+。
   - 优缺点：GQA/MQA几乎无损压缩KV缓存2~8倍；PagedAttention显存浪费减少50%以上；但滑动窗口可能丢失远距离依赖。
   - 工程实践：vLLM的PagedAttention已成为事实标准；Flash Attention 2/3 通过分块+重计算减少HBM读写；Triton/MLU Kernel 定制实现更高效。
   - 历史演变：从标准MHA→MQA→GQA→PagedAttention→Shadow KV，2025年出现了预测式KV缓存（Predictive KV Cache），通过预计算常驻前缀来加速。

5. **综合性工程实践：统一调度与边缘优化**
   - 原理：将上述技术组合进推理框架，同时引入动态批处理（Continuous Batching）、Prefill/Decode分离、弹性并行（TP/PP/EP）和异步调度，最大化GPU利用。
   - 应用：所有主流大模型推理服务（ChatGPT、Claude、开源vLLM/SGLang/TGI）均采用此类架构。
   - 优缺点：系统性优化可带来10x+吞吐量提升，但框架部署复杂度增加；多层调度带来的调度延迟需被掩盖。
   - 工程实践：NVIDIA TensorRT-LLM 中集成FP8量化+IPEX+FasterTransformer；华为昇腾昇思MindSpore针对CANN优化。
   - 对比：单一技术只能解决局部瓶颈，系统工程化（如vLLM+PagedAttention+AWQ+Spec Decode）才能达到服务级吞吐量最优。
